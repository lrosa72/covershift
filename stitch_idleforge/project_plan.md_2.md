# 枇杷的挂机工坊 (Pipa's IdleForge) - 核心项目策划

**版本**：v1.0
**日期**：2026-03-22
**状态**：待评审

## 执行摘要

### 项目定位
这不是一个简单的AI工具——这是**个人生活操作系统**，是一座充满人情味的「数字领地」。

### 核心愿景
将人类的瞬时灵感（低熵）通过Agent密集计算（过程熵）转化为高质量工程产物（高价值低熵）的转换器。

### 世界观设定
- **枇杷**：工坊主/看板娘，微笑接待委托
- **Moss**：核心动力炉/管家，幕后调度
- **工人们**：各工位专业AI小人
- **世界大树**：成果变成树上的果实

---

## 1. 问题分析

### 核心痛点
1. **想法太多，时间太少**：大量灵感无法落地
2. **AI容易猛干**：缺乏人类意图对齐，产出不符合预期
3. **经验无法积累**：每次都是从零开始
4. **等待焦虑**：不知道AI在做什么

### 解决方案
- **人类共鸣层**：关键节点停下来确认
- **长期记忆系统**：历史经验复用
- **游戏化可视化**：看到进度，缓解焦虑
- **任务分拣**：不同类型任务用不同流水线

## 2. 核心功能设计

### 2.1 任务分拣中心

#### 四种工单类型

| 类型 | 颜色 | 范例 | 处理方式 | 产物 |
|------|------|------|---------|------|
| **现实锚点** | 🔴 红色 | 佳福大厦办税务、希腊电表 | Assistant Agent | Notion Action List |
| **虚拟工厂** | 🔵 蓝色 | AI漫剧、RVC复刻、猫猫视频 | 多媒体流水线 | 视频/图像文件 |
| **逻辑实验室** | 🟢 绿色 | 量化系统、Python脚本 | 代码沙箱 | Git代码+报告 |
| **知识内化** | 🟡 黄色 | 吴恩达课程、听播客 | Summary Agent | 个人知识库 |

### 2.2 人类共鸣层

#### 首席架构师访谈
- 想法进入队列后，不直接开工
- 5个问题深度对齐意图
- 生成3个Lo-Fi技术方案供选择
- 熵值>90%拒绝下发

#### 技术树机制
- 决策锚点：关键选择处闪烁问号
- 多方案并行：同时跑不同分支对比
- 一键回溯：点树节点回滚

#### 共鸣检查协议
- 阶段性明信片汇报
- 思维链查看器：随时看Agent在想什么
- 直接干预：输入指令，Agent立即调整

### 2.3 长期记忆系统

#### 三层存储架构

| 层级 | 技术 | 内容 |
|------|------|------|
| **情节记忆** | PostgreSQL | 完整任务记录、决策链 |
| **语义记忆** | ChromaDB | 跨项目知识、代码片段 |
| **程序记忆** | PostgreSQL+Chroma | 避坑指南、工作流模版 |

#### 记忆提取流程
```
任务开始前
    ↓
提取关键词
    ↓
检索类似任务的成功/失败案例
    ↓
生成上下文摘要
    ↓
注入Agent的prompt
```

### 2.4 质量评分模型

#### 四个维度（加权总分）

| 维度 | 权重 | 评分者 |
|------|------|--------|
| 可行性 | 25% | Researcher |
| 代码质量 | 25% | Reviewer + tree-sitter |
| 功能达成率 | 30% | Tester |
| 演化潜力 | 20% | Orchestrator |

**阈值**：≥80分才能合并入主干

### 2.5 Moss 大脑

#### 核心能力
- **异常检测**：死循环、积压、互相推诿
- **降维干预**：拆解任务、强制协作、基因改造
- **资源配置**：根据负载自动选择模型（Flash/Pro/Ultra）
- **策略演化**：观察Agent组合效率，自动调整参数
- **全景广播**：冷酷电子音实时播报

#### 冷启动协议
系统停滞后：
1. Moss接管，暂停所有任务
2. 总结矛盾点，生成冲突报告
3. 临时创建Decision Agent
4. 生成3个备选方案
5. 强制执行，打破僵局

## 3. 技术架构

### 3.1 技术栈

| 层级 | 技术选型 |
|------|---------|
| Web框架 | FastAPI |
| 任务队列 | Celery + Redis |
| 关系数据库 | PostgreSQL |
| 向量数据库 | Chroma (开发) / Pinecone (生产) |
| 缓存/KV | Redis |
| 前端 | React + PixiJS + Framer Motion |
| LLM框架 | LangChain |
| 代码分析 | tree-sitter |
| 容器化 | Docker |
| 版本控制 | Git |

### 3.2 核心组件

```
idleforge/
├── api/              # FastAPI接口
├── core/             # 编排器、记忆、评分、Moss
├── agents/           # 7种专用Agent
├── models/           # 数据模型
├── protocols/        # 消息、工作流协议
├── services/         # Notion、LLM、Git客户端
└── tools/            # 代码执行、tree-sitter
```

## 4. 进化路线图

### 1.0 阶段：功能骨架 (Month 1-2)

**目标**：建立闭环，实现「想法入，结果出」

**核心交付**：
- ✅ FastAPI + Celery编排器
- ✅ 5个基础Agent（Researcher/Designer/Developer/Tester/Reviewer）
- ✅ Notion API双向同步
- ✅ Docker沙箱环境

**成功标准**：
- [ ] 能通过API提交想法
- [ ] 想法能流转完5个Agent
- [ ] 结果能同步到Notion
- [ ] 生成的代码能在沙箱运行

### 2.0 阶段：生命工厂 (Month 3-5)

**目标**：引入「灵魂」——游戏化交互、长期记忆与质量评分

**核心交付**：
- ✅ React游戏化可视化界面（工厂平面图）
- ✅ 三层长期记忆系统
- ✅ 多维评分模型
- ✅ 人类干预码头（HITL）
- ✅ 首席架构师Agent
- ✅ 技术树机制

**成功标准**：
- [ ] 可视化界面能看到Agent工作
- [ ] 类似想法能复用历史经验
- [ ] 评分系统能正确过滤低质量结果
- [ ] 人类能在关键节点干预
- [ ] 至少3个迭代周期的记忆积累

### 3.0 阶段：自治生态 (Year 1+)

**目标**：实现「自我进化」与「跨界融合」

**核心交付**：
- ✅ Moss强人工智能大脑（资源配置权）
- ✅ Agent进化机制（竞争、协作、遗传、变异）
- ✅ 多源自动触发（arXiv、GitHub热点自动生成想法）
- ✅ 开源社会化网络（分布式AI实验室）
- ✅ 跨界共鸣（金融与创作联动、教育与项目进化）

**成功标准**：
- [ ] Moss能自动调整资源配置
- [ ] Agent能通过经验提升表现
- [ ] 系统能自动生成有价值的想法
- [ ] 支持多人协作模式
- [ ] 形成一个小型的自治生态

## 5. 核心场景演示

### 场景1：AI漫剧工厂（1.0实现）

```
1. 你丢入：「开始生成深夜便利店」
   ↓
2. Architect访谈：「是否沿用Pipa人设？」
   ↓
3. 你确认
   ↓
4. Designer：调用Pixverse/即梦做风格迁移
   ↓
5. Moss：启动n8n工作流
   ↓
6. 流水线：脚本→图像→动画→音频→剪辑
   ↓
7. 共鸣检查：「这符合你心目中的氛围吗？」
```

### 场景2：希腊电表（2.0私有上下文）

```
1. 你丢入：「修改希腊电表」
   ↓
2. Moss自动检索：
   ✅ Eurobank邮件往来
   ✅ 房屋合同(FRAGIADON 16-18)
   ✅ 过往账单
   ↓
3. Assistant自动填充表格
   ↓
4. 生成办事清单+路线规划
```

### 场景3：金融与创作联动（3.0跨界共鸣）

```
1. Moss监测：AI游戏板块股票大涨 +15%
   ↓
2. 检测到：你有AI漫剧项目在进行
   ↓
3. Moss建议：「增加AI漫剧项目token预算+30%」
   ↓
4. 自动执行：预算调整 + 相关新闻加入背景
```

## 6. 风险与应对

| 风险 | 影响 | 概率 | 应对策略 |
|------|------|------|---------|
| 1.0 scope膨胀 | 高 | 中 | 严格MVP定义，定期裁剪 |
| API成本超支 | 高 | 高 | 预算控制+模型降级策略 |
| 记忆检索效果差 | 中 | 中 | 从小规模开始，迭代优化 |
| 游戏化开发慢 | 中 | 高 | 分阶段：先简单UI，再视觉效果 |
| 3.0过于复杂 | 低 | 中 | 保持插件化，核心轻量化 |
